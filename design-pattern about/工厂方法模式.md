# 工厂方法模式

父类中提供一个创建对象的方法， 允许不同子类决定实例化对象的类型。

为了提供代码结构的扩展性，屏蔽每一个功能类中的具体实现逻辑。让外部可以更加简单的只是知道调用即可，同时，这也是去掉众多`ifelse`的方式。

左侧使用设计模式后， 不同类型的业务放到了单一的service中。
